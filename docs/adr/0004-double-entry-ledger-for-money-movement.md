# 0004. 모든 자금 이동은 복식부기 원장으로

- **Status**: Accepted
- **Date**: 2026-06-18

## Context

투자 예치, 대출 실행, 상환, 정산 등 모든 자금 이동에서 "돈이 생기거나 사라지지 않음"을 보장해야 한다.
단순히 각 계정의 잔액을 더하고 빼는 방식은 버그가 끼면 정합성이 조용히 깨진다.

## Decision

회계의 **복식부기(double-entry bookkeeping)**를 도메인 모델로 채택한다.
모든 자금 이동은 `LedgerTransaction`으로 표현되며, 생성 시점에 **차변 합계 == 대변 합계** 불변식을 강제한다.
이 불변식을 통과하지 못하는 거래는 애초에 객체로 만들어질 수 없다.

## Consequences

- **긍정**: "돈의 보존 법칙"이 타입 수준에서 강제된다. 원장만 추적하면 자금 흐름 전체를 감사(audit)할 수 있다.
- **부정/비용**: 단순 잔액 업데이트보다 모델이 무겁고, 분개를 항상 짝으로 작성해야 한다.
- **후속**: 원장을 append-only로 영속화하고 잔액은 분개 합으로 도출(또는 스냅샷)한다(Phase B2).

## Alternatives considered

- **계정별 단순 잔액 증감** — 구현은 쉽지만 정합성 오류를 구조적으로 막지 못함.
- **이벤트 소싱 전면 도입** — 강력하지만 학습/운영 비용이 큼. 원장 자체가 이미 사실상의 거래 로그이므로 현 단계에선 과함.
